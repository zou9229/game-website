# SEO 套利方法论

## 核心原理

SEO 套利的本质是**时间差**：

```
游戏爆了 → 玩家开始 Google 搜索 "XXX game" 
→ 但针对这个游戏的攻略/介绍页极少
→ 你抢先建站 → 排名靠前 → 截流搜索流量
→ 流量稳定 → 持续获取被动访问
```

---

## 三类套利窗口

### 1. 新品突发（最佳窗口）
- 游戏刚上线就病毒传播
- 窗口期：1-7 天
- 特征：Twitter 突然刷屏、Discord 服务器爆满
- 典型案例：Slither.io, Paper.io, Zombs.io

### 2. 老游新火（有效窗口）
- 游戏已存在，但被大主播带火
- 窗口期：3-14 天
- 特征：YouTube 相关视频突然爆发
- 典型案例：Among Us（2020爆火）

### 3. 梗文化扩散（意外窗口）
- 某个游戏机制变成网络梗
- 窗口期：7-21 天
- 特征：TikTok/微博出现大量二次创作
- 典型案例：各种 "brainrot games"

---

## SEO 潜力判断流程

```
Step 1: 发现游戏
    ↓
Step 2: 验证搜索量
    搜索 "[游戏名]" 
    看 Google 自动补全有没有 > 100/月
    ↓
Step 3: 验证内容竞争
    搜索 "[游戏名] guide" 或 "[游戏名] tips"
    结果 < 20 个 = 低竞争
    结果 > 100 个 = 高竞争
    ↓
Step 4: 评估流量质量
    看 Google 结果里有没有：
    - 官方 wiki（有 = 流量精准）
    - 纯广告页（有 = 商业价值高）
    ↓
Step 5: 确定内容类型
    根据游戏类型决定建什么页：
    - 多人竞技 → 攻略 + 服务器/代码
    - 休闲益智 → 玩法大全 + 通关技巧
    - 恐怖解谜 → 剧情解析 + 攻略
```

---

## 内容类型优先级

| 类型 | 吸引力 | 制作难度 | 竞争度 |
|------|--------|---------|--------|
| 攻略/通关 | ⭐⭐⭐⭐ | 低 | 高 |
| 视频合集 | ⭐⭐⭐⭐⭐ | 低 | 低 |
| 代码/礼包 | ⭐⭐⭐⭐ | 低 | 低 |
| 评测/试玩 | ⭐⭐⭐ | 中 | 中 |
| 剧情解析 | ⭐⭐⭐ | 中 | 低 |
| 开发者访谈 | ⭐⭐ | 高 | 极低 |

---

## 判断标准速查

**✅ 立即行动：**
- 发布 < 7 天 + 搜索量上升中 + 内容页 < 10 个
- 被大主播提及 + 社交媒体讨论增长

**⚠️ 谨慎行动：**
- 发布 7-30 天 + 搜索量稳定 + 内容页 10-50 个
- 已有几个 wiki/攻略站但质量不高

**❌ 不值得做：**
- 发布 > 30 天 + 内容页 > 100 个
- 游戏过于小众（月搜索 < 100）
- 版权问题（品牌词被注册）
